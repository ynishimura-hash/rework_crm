# Rework 顧客管理システム 仕様書

## 概要
Rework事業専用の独立した顧客管理システム。EISのメインシステムとは異なる固有の営業プロセス・データ構造（Excel管理由来）に合わせて最適化されています。

## データソース
- 基本データ: Rework営業管理DB（Excel）由来のJSON
- ファイルパス: `src/data/rework/mock_db.json`

## 固有のデータ構造と機能仕様
1. **企業リスト（取引先）とサービス構造**
   - EISの標準構造とは異なり、「提案サービス詳細」（AI定額制研修、DX定額制研修など）と企業が強く紐付く。
   
2. **商談・議事録管理**
   - 「契約書/計画届」等のRework特有のステータス管理。
   - 人数と単価をもとにした「見積金額」「見込み予測金額」の算出機能。
   - 議事録テーブルを通じた履歴の紐付け。

## 独立性とEISとの連携（開発方針）
1. **完全分離と別データベース（Supabase）の利用**
   - 現状はローカルJSONデータ＋一部をEISと同環境にてテストしていますが、**最終的にはRework社様専用のSupabase（データベース）で環境を構築して稼働**させます。
   - `NEXT_PUBLIC_SUPABASE_URL_REWORK` といった環境変数を導入し、ソースコード内でRework環境の場合は接続先DBを相手方のSupabaseに向けるような設計（もしくはフロントエンドごとビルドを分離して相手方サーバーにデプロイする設計）にて進めます。

2. **EISからの機能取り込み**
   - EISで開発される「名刺AI読み取り」や「UIの改善」をReworkに実装する際、この仕様書を元にデータマッピング（ReworkのJSON構造への変換）を追加するだけで即座に適用できるようにします。
